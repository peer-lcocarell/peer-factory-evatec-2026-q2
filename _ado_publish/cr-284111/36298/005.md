# Test Case Title
284111 Self-Service Box - New Box Self-Service WBS Auto-Generation and Save (Happy Path)

## Requirement
- CR: 284111
- Ticket: 34682
- Requirement ID: 36298
- Requirement: When Self-Service is enabled, WBS is visible, required, auto-generated from Work Order, and saved.

## Preconditions

- User is signed in with `Boxes_Create` and `Boxes_View` permissions.
- At least two active Work Orders exist with distinct external IDs.
- User is on `Boxes > Create New Box`.

| Step | Action | Expected Result |
|------|----------|----------------|
| 1 | Start New Box wizard and complete mandatory non-WBS fields. | User can proceed to self-service/WBS section. |
| 2 | Select `Is the box from the Self-Serve Station? = Yes`. | `WBS Element` field becomes visible and required. |
| 3 | Select Work Order A. | `WBS Element` auto-populates as `{ExternalIdA}-##-##`. |
| 4 | Change to Work Order B. | `WBS Element` updates to `{ExternalIdB}-##-##`. |
| 5 | Complete remaining required fields and save. | Save succeeds and confirmation is shown. |
| 6 | Open saved box in view mode. | `Is Self-Serve = Yes` and `WBS Element` is displayed with saved value. |
| 7 | Return to list and locate saved box. | Box is listed with `Self-Service = Yes`. |

## Expected Outcome

Self-service New Box flow auto-generates WBS from selected Work Order, enforces required behavior, and persists values correctly.
