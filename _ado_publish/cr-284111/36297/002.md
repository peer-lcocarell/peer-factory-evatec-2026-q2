# Test Case Title
284111 Self-Service Box - Boxes List Status Multi-Select and URL Persistence (Happy Path)

## Requirement
- CR: 284111
- Ticket: 34682
- Requirement ID: 36297
- Requirement: Status filter supports multi-select and filter state persists in URL.

## Preconditions

- User is signed in with `Boxes_View` permission.
- Data contains at least three statuses with known matching records.
- User is on the `Boxes` list page.

| Step | Action | Expected Result |
|------|----------|----------------|
| 1 | Open Status filter dropdown. | Status options are displayed and selectable. |
| 2 | Select two statuses (example: Active, Removed). | Grid updates to rows matching either selected status. |
| 3 | Verify URL query parameters after filtering. | URL contains status filter state for both selected values. |
| 4 | Copy URL and open it in a new browser tab. | Same filtered result set is loaded. |
| 5 | Use browser refresh on filtered list. | Selected statuses remain applied; result set remains consistent. |
| 6 | Clear one selected status and keep one selected. | Grid updates to the remaining status only; URL updates accordingly. |
| 7 | Clear all status filters. | Grid returns to default set; status parameters are removed/reset in URL. |

## Expected Outcome

Status multi-select filtering works consistently and URL fully preserves/restores filter state.
