# R10.2 Update for CR 286836 - Edit ERP Hardware Component: Verify the tester can update hardware component details with valid data (Happy Path)

## Requirement
CR: 286836
Ticket: 34948
Requirement ID: 75372
Requirement: Edit ERP hardware component (Hardware Components)

## Preconditions

- Tester is signed in with permissions to view and edit Hardware Components.
- At least one active ERP-imported hardware component exists.
- Hardware Components page is accessible.
- At least one editable comment field and the `Deleted` toggle are visible for the selected ERP component.

| Step | Action | Expected Result |
|------|----------|----------------|
| 1 | Open Hardware Components and search for an existing ERP hardware component. | Matching component is displayed and can be selected. |
| 2 | Open the component in Edit mode and note baseline values for comment and `Deleted`. | Edit screen opens with the comment field displaying the current saved value and `Deleted` toggle reflecting the current saved state. Tester records both values. |
| 3 | Enter a new valid value in the comment field. | The comment field accepts the new value and the change is reflected in the field. |
| 4 | Toggle the `Deleted` field to the opposite state from the baseline value recorded in step 2. | The `Deleted` toggle updates to the new state and Save is enabled. |
| 5 | Click Save. | Save succeeds without validation errors and success feedback is displayed. |
| 6 | Re-open the same component in Edit mode. | Comment field displays the value entered in step 3 and `Deleted` toggle reflects the state set in step 4. |
| 7 | Open component from list/details view. | Comment and `Deleted` columns in the list and details view display the values saved in steps 3 and 4. |

## Expected Outcome

The tester verifies that ERP hardware component comment and `Deleted` values are updated with valid input and that saved changes persist across Edit and list or detail views.
