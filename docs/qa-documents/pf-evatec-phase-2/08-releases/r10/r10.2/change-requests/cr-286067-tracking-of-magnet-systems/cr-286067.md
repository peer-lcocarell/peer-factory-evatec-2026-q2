# CR 286067 - Tracking of Magnet Systems in Evatec Fabric

## Overview

- Change Request: 286067
- Ticket: 35008
- Owner: Nicola Basley
- Release: R10.2
- Area Path: PFEvatec/WF06 (Logbook)/Magnet Systems
- Iteration Path: PFEvatec/Phase 2/R10/R10.2

## Purpose

Track individual magnet systems in Evatec Fabric in the same way targets are tracked.

Each magnet system must:

- Be represented as a unique physical asset.
- Have an immutable serial number.
- Maintain lifecycle history.
- Support checkout, installation, return, retirement, and prototype management.
- Be traceable independently from article number.

## Business Decisions

### Magnet System Types

- Created only from ERP Article Number.
- Manual Magnet System Type creation is not allowed.
- Multiple Magnet Systems can exist for the same prototype type.
- Modifying a prototype creates a new prototype variant.

### Magnet Systems

- Created only from an existing Magnet System Type.
- Manual Magnet System Type definition is not allowed during creation.

### Checkout and Installation

- Checkout and installation are separate actions.
- Checkout records user and tool in Magnet System history.
- Installation is optional after checkout.
- Installation, when done, is logged in both Magnet System history and Tool Logbook.

### Inspection Notifications

- Monthly inspection emails are not required.
- Users can filter by inspection flag and location.

### Hardware Configuration Role

- A dedicated Magnet System role type is required.
- Implementation follows the existing Target role model.

### Inspection Required Flag

- Informational only.
- No current operational restriction.

## Functional Scope

### Magnet System Types

- Create from ERP articles.
- View and edit.
- Upload and manage documents.
- View history.
- Define prototype type as standard.

### Magnet Systems

- Create, view, edit.
- Upload and manage documents.
- View history.
- Checkout and return.
- Retire.
- Modify.
- Print label.

### Hardware Configuration

- Magnet System assignment and replacement.
- Learned associations.
- Hardware configuration export.
- Module copy operations.
- Configuration history.

### Logbook

- Magnet System install and uninstall entries.
- Removal of related entries.
- Traceability between Logbook and Magnet System history.

## Core Business Rules

### Magnet System Identity

- Magnet System Number format: MS0001.
- Serial number is required.
- Serial number is immutable.
- Magnet System Number is immutable.

### Status and Location

- In Use -> Tool name.
- In Stock -> Empty.
- In Inspection -> Empty.
- Retired -> Empty.

### Supported Actions by Status

- In Stock -> Check Out, Retire.
- In Use -> Return.
- In Inspection -> Return, Retire.
- Retired -> None.

### Authorization

Lifecycle actions are limited to:

- Administrator
- Logistics

### Prototype Rules

ECL rules:

- All Magnet Systems have serial numbers.
- Modified standard Magnet Systems become prototypes.
- Modification affects internal magnet arrangement only.
- Multiple instances may use the same prototype type.

Fabric rules:

- Magnet System identity does not change.
- Prototype changes are represented through Magnet System Type changes.
- Variant identifiers are used for prototypes (for example V01, V02, V03).
- Prototype inherits attributes from originating type.
- Description captures modification details.
