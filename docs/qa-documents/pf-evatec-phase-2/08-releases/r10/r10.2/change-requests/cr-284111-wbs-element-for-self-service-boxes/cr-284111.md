# CR 284111 – WBS Element for Self-Service Boxes
https://ops1.peergroup.com/DefaultCollection/PFEvatec/_testPlans/define?planId=291616&suiteId=292237

## High-Level Summary

CR 284111 introduces support for tracking boxes that originate from a self-service station and capturing a corresponding WBS (Work Breakdown Structure) Element for accounting and project cost tracking.

The change adds new box attributes, validation rules, list enhancements, and filtering capabilities. Users can identify a box as originating from a self-service station during creation or editing.

When selected, a WBS Element becomes required and is automatically generated using the associated Work Order number.

## Business Value

- Improves accounting and project cost tracking for self-service boxes.
- Captures WBS information during box creation and maintenance.
- Reduces manual entry through automatic WBS population.
- Improves visibility of self-service boxes through list views and filtering.
- Provides improved reporting and auditing capabilities.

## Requirement 36298 – New Box

### Difference Introduced by the Change

The New Box workflow now supports Self-Service boxes and WBS Element management.

### New Field: From Self-Serve Station

A new field has been added after Work Order selection.

#### Field Type

- Yes/No toggle

#### Purpose

- Identifies whether the box originates from a self-service station.

#### Default Behavior

- Defaults to Yes when the box is identified as a self-service box.

### New Field: WBS Element

A new WBS Element field has been introduced.

#### Visibility

Displayed when:

- From Self-Serve Station = Yes

#### Validation Rules

Required when:

- From Self-Serve Station = Yes

Maximum length:

- 60 characters

Required format:

```text
[WorkOrderNumber].##.##
```

### Automatic Generation Rules

When a Work Order is selected:

- The Work Order number becomes the WBS prefix.
- Invalid or incorrectly formatted values are regenerated automatically.

Generated format:

```text
[WorkOrderNumber].00.00
```

### Impact

The requirement now includes:

- Self-Service indicator
- WBS Element field
- WBS validation rules
- WBS auto-generation logic

### QA Considerations

#### Self-Service Toggle

Verify:

- Toggle is displayed.
- Default behavior is correct.
- Values persist after save.

#### WBS Visibility

Verify:

- WBS Element is displayed when Self-Service = Yes.
- WBS Element is hidden or disabled when Self-Service = No.

#### WBS Validation

Verify:

- Required field enforcement.
- Maximum length validation.
- Format validation.
- Invalid values cannot be saved.

#### WBS Auto-Generation

Verify:

- WBS is generated from the selected Work Order.
- Invalid values are regenerated correctly.

Example:

```text
WO12345.00.00
```

#### Work Order Updates

Verify:

- Changing the Work Order updates the WBS prefix.
- Generated values remain valid.

#### Save Validation

Verify save is blocked when:

- Self-Service = Yes
- WBS Element is empty

Verify save is blocked when:

- Self-Service = Yes
- WBS Element format is invalid

Verify save succeeds when:

- Self-Service = Yes
- WBS Element format is valid

#### Persistence

Verify:

- Self-Service value is saved correctly.
- WBS Element value is saved correctly.
- Values appear consistently across Create, Edit, View, and List screens.
