### CR 290161 – Error: Use PDT to Create the Tool Definition for New Tool Type

**High-Level Summary**

This CR resolves an issue that prevented users from creating tool definitions for certain new tool types in Fabric. When attempting to generate a Tool Definition from Host Interface Documents (HDI), the process failed with a message instructing users to use PDT instead.

The root cause was related to how the system handled new module naming conventions. The tool contained both PM1 and PM1 alias values, resulting in duplicate matches during lookup operations and causing tool definition creation to fail.

### Business Value

* Enables successful creation of tool definitions for new equipment types.
* Supports onboarding of new tool configurations without manual workarounds.
* Prevents tool setup delays for customers introducing new module naming structures.
* Improves compatibility with evolving equipment configurations.

### Key Areas Impacted

* Tool Definition creation
* PDT integration workflow
* Module name and alias handling
* New tool type setup/configuration

### QA Focus

* Verify tool definitions can be generated successfully for new tool types.
* Confirm module aliases and naming conventions are handled correctly.
* Validate duplicate lookup scenarios no longer cause failures.
* Perform targeted regression around Tool Definition creation workflows.

### Risk Level

**Low**

The change is targeted to a specific lookup issue involving new module names and aliases. Customer validation has already been completed, and the fix was released as a patch to unblock an immediate deployment need.
