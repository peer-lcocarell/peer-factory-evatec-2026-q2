# CR 285729 – Search HW Components

## High-Level Summary

This CR introduces a new **Hardware Search** feature that helps users quickly locate where hardware components are installed within the factory configuration.

Users can search using:

- Part Number
- Description
- Serial Number
- Installation date range
- Docked or undocked module scope

Search results display the associated **Tool** and **Module** where the hardware is installed and provide navigation links that allow users to open those locations directly.

## Business Value

- Simplifies locating hardware across tools and modules.
- Reduces time spent manually reviewing hardware configurations and logbook records.
- Supports both current and historical hardware installation tracking.
- Improves troubleshooting and equipment management.

## Key Areas Impacted

- Hardware Search dialog
- Tools page
- Modules page
- Tool Overview
- Hardware installation history
- Logbook data

## QA Focus

- Search accuracy across all supported filters.
- Historical date-range searches.
- Current versus historical installation results.
- Tool and Module navigation links.
- Correct handling of module and submodule locations.

## Risk Level

**Medium**

The feature primarily introduces new search and reporting functionality but relies on historical hardware installation records and module hierarchy data. Incorrect data relationships could result in inaccurate search results or location reporting.

# Requirement Details

## Requirement 36315 – Tool Overview

- State: Resolved (Reintroduced in Scope)
- Assigned To: Development Team

### Change Summary

The Tool Overview page now exposes a **Search Hardware** button alongside the existing Hardware button. The new button is available when the selected tool has a logbook configured.

### Security Rights

- `ToolOverview_View` — access the Tool Overview screen
- `ToolDetails_View` — view Tool Details and History
- `HardwareConfiguration_View` — view Hardware and Search Hardware actions
- `Logbook_View` — view the Logbook for a tool

### New Capabilities

- Tool list now shows: name, tool type, owner, connection type, substrate size, position, production work order, equipment number, live status, and E10 state.
- Tool type, connection type, and status can be filtered by multiple values.
- **Search Hardware** button added alongside the existing Hardware button.

### QA Considerations

- Tool links from Hardware Search results open the correct Tool Overview page.
- Tool information displayed in search results is accurate.
- Historical search results reference the correct tool assignment.
- Search Hardware button is only visible when `HardwareConfiguration_View` right is held and the tool has a logbook configured.

## Requirement 226372 – Modules

- State: Active
- Assigned To: Development Team

### Change Summary

The Modules screen now includes Hardware Search access. The module list has been extended with additional columns, and the Hardware Search action has been added as a module-level action.

### Security Rights

- `Modules_View` — access the Modules screen
- `Modules_Create` — create or copy a module
- `Modules_Edit` — edit a module
- `Modules_Remove` — retire a module
- `Modules_Admin` — permanently remove a module
- `HardwareConfiguration_View` — view Hardware Configuration and access Hardware Search
- `Logbook_View` — view the Logbook for a module

### New Capabilities

- Module list now includes: module number, serial number, name/label, module type, location, status, article number, description, and submodule count.
- **Search for hardware components** action added — requires `HardwareConfiguration_View`.
- Location field for non-Retired real modules links to the tool and module location when docked.

### QA Considerations

- Module links from Hardware Search results open the correct module.
- Submodule locations are reported correctly.
- Docked and undocked module filtering behaves as expected.
- Historical installation records reference the correct module location.
- Hardware Search action is visible only when `HardwareConfiguration_View` right is held and the module type has a logbook configured.
