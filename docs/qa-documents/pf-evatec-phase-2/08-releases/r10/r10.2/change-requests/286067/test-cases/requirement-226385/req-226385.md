13.7.3REQUIREMENT: Copy Module Hardware Configuration
The user can select to copy the hardware configuration from another module with the same module type.

PF will use the current hardware configuration version from the source module to set the pending values.

1.Text & File roles will not be copied

2.If copying from a real module

a.to another real module any installed target or magnet system will not be copied

b. to abstract module any installed target will be converted to the target type and installed magnet system will be converted to the magnet system type

3.If copying from an abstract module to a real module

a.Wildcards will not be copied

b.Target roles will not be copied

c.Magnet system roles will not be copied

d.Roles with multiple values will show the same “Multiple hardware components found” validation error as ‘BOM import’, preventing save until there is only one

e.Roles requiring serial numbers will show the validation error preventing save until it is available

Once there is at least one pending change and all validation errors have been cleared the user can save the hardware configuration for the module.

Users must have the HardwareConfiguration_Edit security right to perform these actions.
