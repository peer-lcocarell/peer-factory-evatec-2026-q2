# CR 286836 – Article Import

## High-Level Summary

This CR enhances the Hardware Component (Article) import process by introducing support for configurable import formats and improving compatibility with updated PLM export files.

The change removes dependence on a single import structure, allowing users to import hardware component data from different file formats without requiring application changes or application restarts.

Additionally, the CR addresses import processing issues by:

- Handling status values correctly.
- Preventing valid Hardware Components from being unintentionally deactivated when importing partial article lists or BOM-driven datasets.

## Business Value

- Supports changing PLM export formats without requiring development changes.
- Provides a more flexible and maintainable import framework.
- Reduces manual effort required to prepare import files.
- Prevents accidental deactivation of valid Hardware Components.
- Improves data integrity and reliability of Hardware Component imports.

## Key Areas Impacted

- Hardware Component Import
- ERP Import functionality
- BOM Import functionality
- Work Order Import functionality
- Import format configuration and mapping
- Hardware Component maintenance

## QA Focus

- Validate configurable import formats and mappings.
- Verify Hardware Component imports using legacy and new file formats.
- Verify correct field mapping and status value handling.
- Confirm partial imports do not incorrectly deactivate existing components.
- Perform regression testing on shared import framework functionality across ERP, BOM, and Work Order imports.

## Risk Level

**Medium**

The change affects a shared import framework used by multiple business processes. Incorrect mappings or import logic could introduce data integrity issues, impact existing import workflows, or lead to unintended Hardware Component activation or deactivation behavior.

# Requirement Details

## Requirement 75372 – Edit ERP Hardware Component

### Difference Introduced by the Change

New functionality was added for users with the **ERP_Edit** security right.

Users can now:

- Select one or more Hardware Components.
- Toggle the value of the **Deleted** column.

This functionality was not available in previous versions.

### Unchanged Behavior

- Users can modify imported ERP comments.
- Save persists changes.
- Cancel closes the page without saving.
- ERP_Edit security rights are required.
- The Article # column continues to provide a link to the Edit feature for authorized users.

### Impact

The requirement scope has expanded from editing ERP component comments to also managing the **Deleted** status of Hardware Components individually or in bulk.

## Requirement 264374 – Import Hardware Components from File

### Difference Introduced by the Change

Import behavior was expanded to update an additional object type.

### Previous Behavior

Any **Target Types** defined from an ERP Hardware Component would have the following fields updated:

- Name (English)
- Name (German)
- Status

### New Behavior

Any **Target Types** and **Magnet System Types** defined from an ERP Hardware Component will have the following fields updated:

- Name (English)
- Name (German)
- Status

### Impact

Imports now propagate updates to:

- Target Types
- Magnet System Types

Additional regression testing should verify that Magnet System Type names and statuses are updated correctly during import.

### Unchanged Behavior

- File format mappings
- Product Group filtering
- Import summary reporting
- Security requirements
- Custom file format rules and mappings

## Requirement 272565 – Import Work Orders from File

### 1. Support for Multiple File Formats

#### Previous Behavior

- Only a Project File could be imported.

#### New Behavior

- Work Orders can be imported using multiple file formats.
- A Format selector is available.
- PF defaults to the **Project File** format.

### 2. Field Mapping Changes

#### Previous Mapping

| PF Name | File Column Name |
|----------|------------------|
| Article # | Project |
| Name | Project Name |
| Application Code | Project Type |
| Status | Processing Status |

#### New Mapping

| PF Name | File Column Name |
|----------|------------------|
| Project # | Project |
| Name | Project Name |
| Project Type | Project Type |
| Status | Processing Status |

#### Mapping Updates

- Article # renamed to Project #.
- Application Code renamed to Project Type.
- Documentation now indicates Project Type was previously named Application Code.

### 3. Custom File Formats

#### New Functionality

Users can create and manage custom import formats.

Required fields:

- Unique format name (maximum 50 characters)
- Project # mapping
- Name mapping

Optional fields:

- Project Type
- Status
- Equipment #
- Product Group
- Product Type
- Business Unit

#### Previous Behavior

- Custom file formats were not supported.

### 4. Import Configuration Enhancements

#### New Behavior

Users must select:

- Import Format
- Import File

#### Previous Behavior

Users only selected a Project File.

### Unchanged Behavior

- Closed and Locked statuses continue to import as Closed.
- Import summaries continue to display:
  - Added Work Orders
  - Updated Work Orders
  - Skipped Work Orders
- ERP_View and ERP_Edit security rights remain required.

### Impact

This is a significant enhancement that transitions the functionality from a fixed Project File import process to a configurable import framework that supports:

- Multiple file formats
- Custom mappings
- Additional optional Work Order fields

## Requirement 287451 – Search for Installed Hardware Components

```text
13.8.1REQUIREMENT: Search for Installed Hardware Components
At times the user needs to search to find out where hardware components are currently or have been installed in both tools and modules.

The search criteria includes

Article # (external ID of the hardware component, can be a partial match)

Article Name (English name of the hardware component, can be a partial match)

Serial # (specified during hardware component installation, can be partial match)

For modules select whether to include all or only dockable

A date range for when hardware component was installed (defaulted to the current date)

Once the user searches PF will display the matches found including

Article (# and name)

Location (tool and/or module)

The location will be a link to the appropriate hardware configuration page.

The user can reset and search again if desired.

Users must have the HardwareConfiguration_View security right to access this feature.
```
