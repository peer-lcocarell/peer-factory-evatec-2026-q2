

# CR 286836 - Article Import

## High-Level Summary

CR 286836 addresses Article Import reliability and flexibility issues in Hardware Components and related import workflows.

The change introduces configurable file format support for imports and resolves status value handling issues that were causing import failures and unintended deactivation behavior.

## Business Value

- Supports evolving PLM export structures without requiring code changes or application restart.
- Reduces manual file manipulation and import preparation effort.
- Improves data integrity by preventing incorrect deactivation side effects.
- Improves long-term maintainability by reusing a configurable import framework.

## Key Areas Impacted

- ERP Hardware Component import
- Work Order import
- BOM import
- Import format and mapping configuration
- Hardware Component active or deleted state handling

## Impacts

- 4 requirements are impacted.
- 75372 - Edit ERP hardware component
- 264374 - Import hardware components from file
- 272565 - Import work orders from file
- 264373 - Load BOM from File

## QA Writing Approach

- Test cases are created with Copilot assistance to improve writing speed and formatting consistency.
- Coverage includes happy path, unhappy path, validation, edge case, and regression scenarios.
- Coverage focuses on import format selection, mapping correctness, status handling, and downstream data integrity.

## QA Notes

- Current local suite includes 4 test cases per impacted requirement.
- Latest additions cover bulk `Deleted` toggle, product group filter validation, Closed and Locked status normalization, and large BOM file responsiveness.
- Remaining planned coverage focuses on permissions and advanced custom format management scenarios.

## QA Testing Approach

- Validate custom import formats and mapping rules for Hardware Components, Work Orders, and BOM.
- Validate default format behavior and required field mapping behavior.
- Validate numeric status value handling and resulting import outcomes.
- Validate partial import behavior does not incorrectly deactivate valid Hardware Components.
- Run regression testing on shared import framework behavior across ERP Import, Work Order Import, and BOM Import.

## Potential Issues

- Incorrect mapping can import invalid or misaligned data.
- Shared import framework changes can impact previously stable import paths.
- Partial import execution can incorrectly update active or deleted state.
- Large files can expose performance and responsiveness issues.

## Risk Level

**Medium**

The change touches a shared import framework and multiple functional areas. Primary risks are data integrity, status handling correctness, and cross-feature regression.

# Requirement Details

## Requirement 75372 - Edit ERP hardware component

### Change Summary

- Users with `ERP_Edit` can now toggle the `Deleted` state for one or more Hardware Components.
- Existing comment edit behavior remains in scope.

### QA Considerations

- Validate edit permissions.
- Validate save and cancel behavior.
- Validate `Deleted` state persistence and list visibility consistency.

---

## Requirement 264374 - Import hardware components from file

### Change Summary

- Import supports multiple file formats.
- Default format remains available.
- Import updates linked Target Types and Magnet System Types for name and status.

### QA Considerations

- Validate default and custom mapping behavior.
- Validate product group filtering behavior.
- Validate status handling including numeric value inputs.
- Validate import summary counts for added, updated, and skipped records.

---

## Requirement 272565 - Import work orders from file

### Change Summary

- Work Orders import supports multiple file formats with a selectable format.
- Required mappings are `Project #` and `Name`.
- Optional mappings include `Project Type`, `Status`, and additional fields.

### QA Considerations

- Validate default format and custom format behavior.
- Validate required mapping enforcement.
- Validate Closed and Locked status normalization behavior.
- Validate import summary counts and downstream Work Order availability.

---

## Requirement 264373 - Load BOM from File

### Change Summary

- BOM import supports configurable file formats.
- Default BOM mapping remains available.
- Custom formats require all mappings except `Name (English)`.

### QA Considerations

- Validate default BOM mapping behavior and tree parsing output.
- Validate custom format creation and required mapping validation.
- Validate invalid file and invalid mapping error handling.
- Validate BOM import regression behavior after framework updates.
