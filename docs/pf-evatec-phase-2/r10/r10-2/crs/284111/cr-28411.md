
# CR 284111 – WBS Element for Self-Service Boxes

## Overview

CR 284111 introduces support for tracking boxes that originate from a self-service station and capturing a corresponding WBS (Work Breakdown Structure) Element for accounting and project cost tracking.

The change adds new box attributes, validation rules, list enhancements, and filtering capabilities. Users can identify a box as originating from a self-service station during creation or editing. When selected, a WBS Element becomes required and is automatically generated using the associated work order number.

## Business Value

- Improves accounting and project cost tracking for self-service boxes.
- Captures WBS information during box creation and maintenance.
- Reduces manual entry through automatic WBS population.
- Improves visibility of self-service boxes through list views and filtering.
- Provides more robust box reporting and auditing capabilities.

## Scope of Change

### Box Management

- Add Self-Service indicator to box records.
- Add WBS Element field to box records.
- Persist Self-Service and WBS Element values in the database.
- Add validation and generation logic for WBS Elements.

### Box Creation

- Allow users to indicate whether a box originates from a self-service station.
- Require WBS Element when Self-Service is enabled.
- Automatically generate a WBS Element from the selected work order when appropriate.

### Box Maintenance

- Display and edit Self-Service and WBS Element fields.
- Ensure changes remain synchronized with the associated work order.

### Box List

- Add Self-Service column.
- Add additional box metadata columns.
- Add multi-select Status filtering.
- Add Entry Date range filtering.

## Acceptance Criteria

### Self-Service Box Support

1. Users can identify a box as originating from a self-service station.
2. Self-Service status is stored with the box.
3. Self-Service status is visible wherever box details are displayed.

### WBS Element Support

1. WBS Element is displayed when Self-Service is enabled.
2. WBS Element is required when Self-Service = Yes.
3. WBS Element is not required when Self-Service = No.
4. WBS Element supports a maximum length of 60 characters.
5. WBS Element follows the format:

   ```
   [WorkOrderNumber].##.##
   ```

6. When a work order is selected:

   - The WBS prefix is populated with the work order number.
   - Invalid values are automatically regenerated as:

     ```
     [WorkOrderNumber].00.00
     ```

7. Changes to the work order update the default WBS value appropriately.

### Box List Enhancements

1. Box List displays the required columns.
2. Status filtering supports multiple selections.
3. Entry Date filtering supports date ranges.
4. Filter selections are preserved in the URL.
5. Self-Service values are visible within the list view.

## Requirement 36298 — New Box

### Requirement Change Summary

The New Box workflow now supports Self-Service boxes and WBS Element management.

### New Fields Added

#### From Self-Serve Station

A new field has been added following Work Order selection.

**Field Type**

- Yes / No toggle

**Purpose**

- Identifies whether the box originated from a self-service station.

**Default Behavior**

- Defaults to Yes when the box is identified as a self-service box.

### WBS Element

A new WBS Element field has been introduced.

#### Visibility

Displayed when:

- From Self-Serve Station = Yes

#### Validation

Required when:

- From Self-Serve Station = Yes

Maximum length:

- 60 characters

Required format:

```text
[prefix].##.##
```

### Automatic Generation Rules

When a Work Order is selected:

- The work order number becomes the WBS prefix.
- Invalid or improperly formatted values are regenerated automatically.

Generated value format:

```text
[WorkOrderNumber].00.00
```

### Functional Impact

Previously required fields included:

- Box Name
- Work Order
- Owner
- Location
- Shelf #
- Comment

The revised requirement adds:

- Self-Service flag
- WBS Element field
- WBS validation rules
- WBS auto-generation logic
