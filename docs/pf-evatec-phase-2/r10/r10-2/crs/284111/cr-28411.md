### CR 284111 – WBS Element for Self-Service Boxes

**High-Level Summary**

This CR introduces support for identifying boxes that originate from a self-service station and capturing a corresponding WBS (Work Breakdown Structure) Element for accounting and project tracking purposes.

When creating or editing a box, users can indicate whether the box came from a self-service station. If selected, a WBS Element field becomes available, is automatically populated from the project number, and requires a valid suffix format. The change also adds self-service visibility and filtering enhancements to the Box List.

### Business Value

* Improves accounting and project cost tracking for self-service boxes.
* Captures WBS information directly during box creation and maintenance.
* Reduces manual data entry through automatic WBS value population.
* Provides visibility into self-service boxes through list filtering and reporting.

### Key Areas Impacted

* New Box wizard
* Create Box page
* Edit Box page
* View Box page
* Box List
* Box filtering
* Database model and validation logic

### QA Focus

* Self-Service selection during box creation.
* WBS Element visibility and required-field behavior.
* Default WBS value generation from the project number.
* WBS validation rules and format enforcement.
* Work order changes updating the default WBS value.
* Self-Service column and filtering functionality on the Box List.
* Entry Date range filtering.
* Database persistence and handling of non-self-service boxes.

### Risk Level

**Medium**

The change impacts multiple box management workflows, introduces new validation rules, and adds database fields. Inconsistencies between create, edit, view, and list screens could result in incorrect WBS data or filtering behavior.

**Summary**

Add support for identifying boxes that originate from a self-service station and capture a corresponding **WBS (Work Breakdown Structure) Element** for accounting and project cost tracking.

When creating or editing a box, the user can flag it as coming from a self-service station. When flagged, a **WBS Element** field becomes available, is auto-populated from the associated project number, and is validated against the required suffix format. The Box List is also enhanced with self-service visibility and filtering.

**Business Value**

- Improves accounting and project cost tracking for self-service boxes.
- Captures WBS information at the point of box creation/maintenance.
- Reduces manual data entry via auto-population from the project number.
- Provides visibility into self-service boxes through list filtering and reporting.

**Scope of Change**

- **New Box** wizard – add Self-Service flag and WBS Element field.
- **Create Box / Edit Box / View Box** pages – display, edit, and persist WBS Element.
- **Box List** – new Self-Service column, filter, and Entry Date range filter.
- **Database model** – new fields, validation rules, and migration.
- **Work Order** changes – update default WBS value when the linked work order changes.

**Acceptance Criteria**

1. User can mark a box as **Self-Service** in the New Box wizard, Create Box, and Edit Box pages.
2. When Self-Service is selected, the **WBS Element** field is shown and required.
3. WBS Element auto-populates from the project number and enforces the required suffix format.
4. Changing the linked work order updates the default WBS Element value.
5. WBS Element persists correctly and displays consistently across Create, Edit, View, and List screens.
6. Box List exposes a **Self-Service** column, a Self-Service filter, and an Entry Date range filter.
7. Non-self-service boxes are unaffected (no WBS Element required or stored).

**Out of Scope**

- Changes to non-box workflows that consume WBS data downstream (reporting, exports beyond Box List).

**QA Focus**

- Self-Service selection during box creation.
- WBS Element visibility and required-field behavior.
- Default WBS value generation from the project number.
- WBS validation rules and format enforcement.
- Work order changes updating the default WBS value.
- Self-Service column and filtering functionality on the Box List.
- Entry Date range filtering.
- Database persistence and behavior for non-self-service boxes.

**Risk Level:** Medium — touches multiple box-management workflows, introduces new validation rules, and adds database fields. Inconsistencies between Create, Edit, View, and List screens could result in incorrect WBS data or filtering behavior.
