### CR 286836 – Article Import

**High-Level Summary**

This CR enhances the Hardware Component (Article) import process by introducing support for configurable import formats and improving compatibility with updated PLM export files. The change removes dependence on a single import structure, allowing users to import hardware component data from different file formats without requiring application changes or restarts.

In addition, the CR addresses issues with import processing, including handling status values correctly and preventing valid Hardware Components from being unintentionally deactivated when importing partial article lists or BOM-driven datasets.

### Business Value

* Supports changing PLM export formats without development changes.
* Provides a more flexible and maintainable import framework.
* Reduces manual effort required to prepare import files.
* Prevents accidental deactivation of valid Hardware Components.
* Improves data integrity and reliability of Hardware Component imports.

### Key Areas Impacted

* Hardware Component Import
* ERP Import functionality
* BOM Import functionality
* Work Order Import functionality
* Import format configuration and mapping
* Hardware Component maintenance

### QA Focus

* Validation of configurable import formats and mappings.
* Hardware Component imports using legacy and new file formats.
* Correct field mapping and status value handling.
* Verification that partial imports do not incorrectly deactivate existing components.
* Regression testing of shared import framework functionality across ERP, BOM, and Work Order imports.

### Risk Level

**Medium**

The change affects a shared import framework used by multiple business processes. Incorrect mappings or import logic could introduce data integrity issues, impact existing import workflows, or lead to unintended Hardware Component activation/deactivation behavior.
