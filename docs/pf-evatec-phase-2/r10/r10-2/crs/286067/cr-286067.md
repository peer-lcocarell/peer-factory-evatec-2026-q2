# Executive Summary

This CR introduces a new Magnet System management framework into Fabric. The solution is modeled after existing Target management functionality but includes additional complexities to support prototype lifecycle management, serial-number-based asset tracking, inspection management, hardware configuration integration, logbook integration, and complete audit history.

The feature allows Fabric to track individual magnet systems throughout their lifecycle rather than relying solely on article numbers. Each physical magnet system becomes a unique tracked asset with its own serial number, status, history, location, documentation, and hardware installation records.

This is one of the largest functional additions planned for R10.2 due to its impact across multiple subsystems including:

- Targets framework
- Hardware Configuration
- Logbook
- Database
- Collection Parameters
- Analysis Requests
- Substrate History
- Learned Associations
- Security
- Migration/Upgrade scripts

---

# Business Objective

Evatec requires visibility into:

- All Magnet System Types
- All physical Magnet Systems
- Current locations
- Installed tool assignments
- Inspection status
- Prototype lifecycle changes
- Historical ownership and installation records

The system must support tracking Magnet Systems similarly to Targets while preserving ECL-specific prototype workflows.

# Core Concept

## Magnet System Type

Represents the article-level definition.

Examples:

- Article Number
- ERP Status
- Source Type
- FlexiCat Type
- Applications
- Variant Code
- Successor

A Magnet System Type can have multiple physical Magnet Systems.

---

## Magnet System

Represents a physical asset.

Examples:

- MS0001
- MS0002
- MS0003

Each Magnet System has:

- Unique Magnet System Number
- Serial Number
- Status
- Inspection Date
- Location
- History
- Documents

# Checkout vs Installation

## Checkout

Checkout means:

- Magnet System handed to a user
- User recorded
- Tool recorded
- History entry created

Checkout does NOT install the Magnet System.

---

## Installation

Installation occurs through Hardware Configuration.

Installation must:

- Update Magnet System state
- Create Magnet System History entry
- Create Logbook entry
- Link Logbook entry to Magnet System History

# Prototype Lifecycle

This is the most complex area of the CR.

Unlike Targets, a Magnet System can change its type over time.

Example:

```text
MS0001
Serial #12345

Type A
  ↓
Prototype V01
  ↓
Prototype V02
  ↓
Prototype V03
```

The Magnet System remains the same physical asset.

The type changes.

---

## Prototype Rules

### Rule 1

All Magnet Systems have a serial number.

### Rule 2

Standard Magnet Systems can become prototypes.

### Rule 3

Modification only changes internal magnet arrangement.

### Rule 4

Multiple Magnet Systems can exist for the same prototype.

---

## Prototype Type Creation

When modifying a Magnet System:

### Option 1

### Option 2

Create a new prototype.
